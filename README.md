# Dual-Channel DDS Signal Generator (FPGA-Based)

This project implements a high-precision Dual Channel Direct Digital Synthesis (DDS) system on a Xilinx Artix-7 FPGA (Nexys A7-100T development board). The system generates two independent sine wave outputs with precise frequency, phase, and amplitude control via a UART interface.

## Features

- Dual-channel DDS using Xilinx DDS Compiler IP (version 6.0)
- Independent frequency, phase, and amplitude control for each channel
- UART-based configuration interface for real-time parameter updates
- SPI communication to external DACs (Analog Devices AD5791 or similar)
- On-chip clock generation and management using Clock Wizard and PLL
- Resource-efficient design with streaming axis_scaler blocks for amplitude control
- Built with Xilinx Vivado 2023.2

## Hardware Requirements

- Nexys A7-100T FPGA Development Board (or compatible Artix-7 FPGA board)
- Two DAC modules (e.g., AD5791-based SPI DAC boards)
- USB-UART bridge for configuration (USB-UART on Nexys A7)
- Power supply for DACs (typically ±12V or 0-5V depending on DAC)
- Oscilloscope for observing output signals

### Hardware Connections and Pinout

- **Clock Source**: The system utilizes the Nexys A7's 100 MHz internal oscillator.
- **DAC Interface**: Connected via PMOD ports JA and JB.
- **Pin Assignments** (as defined in Final.xdc):
  - UART_RX (input from host): Pin C4
  - SPI SCLK (to DAC): JA1 / JB1
  - SPI MOSI (to DAC): JA2 / JB2
  - SPI CS (to DAC): JA3 / JB3
  - DAC_LDAC (load DAC): JA4 / JB4
  - DAC_CLR (clear DAC): JA7 / JB7
- Note: JA and JB PMOD connectors are duplicated for Channel A and Channel B respectively.

## Software Dependencies

- Xilinx Vivado Design Suite 2023.2 (or later)
- Optional: SPI flash programming tool (if storing bitstream in flash)

## How to Build

1. **Clone the repository**:
   ```bash
   git clone https://github.com/your-username/Dual-Channel-DDS-Signal-Generator---FPGA-Based.git
   cd Dual-Channel-DDS-Signal-Generator---FPGA-Based
   ```

2. **Restore the Vivado project** using the provided Tcl script:
   - Launch Vivado 2023.2 Tcl Shell (or Vivado IDE and open Tcl Console)
   - Change to the project directory:
     ```tcl
     cd path/to/Dual-Channel-DDS-Signal-Generator---FPGA-Based
     ```
   - Source the restoration script:
     ```tcl
     source deneme4.tcl
     ```
   - This will create a Vivado project named `deneme4` in the current directory.

3. **Generate the bitstream**:
   - In the Vivado IDE, open the `deneme4` project.
   - Generate the bitstream by clicking "Generate Bitstream" in the Flow Navigator.
   - Alternatively, run the following Tcl commands:
     ```tcl
     launch_runs impl_1 -jobs 4
     wait_on_run impl_1
     ```

4. **Program the FPGA**:
   - Connect the Nexys A7 board via USB-JTAG.
   - In Vivado Hardware Manager, program the device with the generated `deneme4.bit` file.

5. **Configure the DDS**:
   - Open a UART terminal (e.g., PuTTY, Tera Term) connected to the USB-UART port of the Nexys A7 (typically 115200 baud, 8-N-1).
   - Send configuration packets to set frequency, phase, and amplitude for each channel.
   - Refer to `packet_parser.v` for the packet format.

## Packet Format (UART Configuration)

The system expects packets to configure the DDS channels. Each packet consists of:
- Start byte: `0xAA`
- Channel ID: `0x00` for Channel A, `0x01` for Channel B
- Register address: 
  - `0x00`: Frequency tuning word (32 bits, little-endian)
  - `0x01`: Phase offset (16 bits, little-endian)
  - `0x02`: Amplitude scaling (16 bits, little-endian, 0-1023)
- Data: Depending on register address (2-4 bytes)
- End byte: `0x55`

Example: Set Channel A frequency to 1 kHz (assuming 100 MHz DDS clock):
- FTW = (1e3 / 100e6) * 2^32 ≈ 42950
- Packet: AA 00 00 96 06 00 00 55 (where FTW=42950 = 0x0000A76E, little-endian: 6E A7 00 00)

## System Architecture

The internal FPGA logic follows a modular pipeline architecture:

1. **UART Receiver and Packet Parser**: Collect incoming serial data byte by byte, analyze packets, and generate valid signals/commands for frequency and phase adjustments to the DDS compiler.
2. **Xilinx DDS Compiler IP**: Generates digital samples of a sine waveform.
3. **AXI Scaler**: Applies digital gain to scale the DDS output to the desired amplitude.
4. **FIFO**: Acts as a clock domain crossing buffer to synchronize processor speed with DAC writing speed.
5. **DAC Controller and SPI Master**: 
   - Send necessary startup configuration commands to initialize the DAC (AD5791).
   - Transmit the data stream alongside a header, adhering strictly to the AD5791 timing diagram.

## Note on Signal Integrity

An earlier version of the design had an inadvertent inversion of the sign bit in the DAC controller (`dac_controller.v`), causing the negative half of the sine wave to be clipped. This issue has been identified and fixed in the current repository. The design now outputs a full sine wave as expected.

If you are using an older version, ensure the following line in `dac_controller.v` does **not** invert the sign bit:
```verilog
m_axis_tdata  = {4'b0001, s_axis_tdata[19], s_axis_tdata[18:0]};
```
